# MIB Beacon 0.6.0

Reliability, accessibility, and cross-platform workflow release of the MIB/SNMP workbench.

## Highlights

- Remote-backed settings and persistent collections now use transaction-safe draft, confirmation, rollback, reconciliation, and stale-response protection.
- Runtime theme normalization enforces WCAG 2.2 AA across bundled and imported themes, including interactive, disabled, focus, and hostile-theme states.
- A central action registry makes keyboard-suitable operations discoverable through the Command Palette with shared confirmations and disabled-state explanations.
- Responsive workspaces now respect measured pane minimums, remain scroll-reachable at short heights, and avoid clipped navigation, packet-console, and trap-composer controls.
- Query, Trap Composer, Live MIBs, and agent groups provide in-context prerequisite creation and automatically resume dependent work.
- Notification preferences and explicit permission controls are available in Settings, with native Android notification support.
- Android chart sharing now writes and shares a real `image/png` cache file instead of a data URL.
- CI and Release enforce a versioned mobile, tablet, desktop, breakpoint-edge, short-landscape, dialog, occlusion, and last-control validation matrix tied to the tested commit.

## Validation note

The local release host had no connected Android device and no installed emulator binary. Native adapter regressions pass locally, and the Release workflow launches the signed APK in an API 35 emulator when APK output is selected.

## Upgrade notes

Back up private application data before replacing an earlier beta. Verify downloaded artifacts against `SHA256SUMS`.
